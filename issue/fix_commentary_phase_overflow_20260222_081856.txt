 FIX: Codex Response 文本+工具调用溢出问题 (Commentary Phase Overflow)
================================================================================

────────────────────────────────────────────────────────────────────────────────
1. 问题描述
────────────────────────────────────────────────────────────────────────────────

Codex 模型在执行工具调用前，会先生成一段 commentary 文本（如 "Using the
openspec-apply-change skill for this request."），然后才发出 function_call 事件。

由于 SSE 是单向流，已发出的 text_delta 事件无法撤回，导致 Claude Code 收到的
响应中同时包含 text blocks 和 tool_use blocks，造成输出混乱。

典型表现：
  - Claude Code 界面出现 "Using the openspec-apply-change skill..." 等非预期文本
  - 文本块与工具调用块混合在同一个响应中
  - 影响对话上下文的清晰度

────────────────────────────────────────────────────────────────────────────────
2. 根因分析
────────────────────────────────────────────────────────────────────────────────

Codex Response API 的 SSE 事件流中，response.output_item.added (type: "message")
携带 phase 字段：

  - phase: "commentary"    → 中间推理/叙述文本，后面通常跟工具调用
  - phase: "final_answer"  → 最终回答文本
  - phase: null/缺失       → 兼容旧版本

codexProxy 此前完全忽略了 phase 字段，将所有文本统一作为 text_delta 输出。

参考来源：
  - Codex CLI 源码: codex-rs/protocol/src/models.rs → MessagePhase 枚举
  - Codex 日志: .codex-log/session.jsonl → 实际携带 phase 字段的事件

────────────────────────────────────────────────────────────────────────────────
3. 解决方案
────────────────────────────────────────────────────────────────────────────────

策略：将 commentary 阶段的文本重定向为 Anthropic thinking blocks。

  phase            | 处理方式
  -----------------+-------------------------------
  "commentary"     | 文本 → thinking_delta (thinking blocks)
  "final_answer"   | 文本 → text_delta (正常文本)
  null / 缺失      | 文本 → text_delta (向后兼容)

优势：
  - commentary 内容保留为 thinking blocks，在 Claude Code 中可见但不污染对话
  - 工具调用与文本块完全分离，不再混合
  - 完全向后兼容：无 phase 字段时行为不变
  - 不修改 instructions.txt（修改会导致上游报错）
